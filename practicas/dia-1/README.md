# Prácticas — Día 1

## Objetivo del día

Crear la cuenta de GitHub, configurar el primer repositorio del portfolio y comenzar a documentar el trabajo técnico usando Markdown.

## Actividades realizadas

- Creación de cuenta en GitHub.
- Creación del repositorio `portfolio-include13`.
- Configuración del repositorio como público.
- Creación del archivo `README.md` principal.
- Edición del README para adaptarlo al proyecto MIDTECH.
- Creación de la carpeta `portfolio/midtech/`.
- Creación del README específico del proyecto MIDTECH.
- Primeros commits documentados con mensajes descriptivos.

## Conceptos trabajados

| Concepto | Explicación |
|---|---|
| Repositorio | Carpeta principal del proyecto con historial de cambios |
| README.md | Archivo principal de presentación del proyecto |
| Markdown | Lenguaje sencillo para escribir documentación técnica |
| Commit | Registro de un cambio realizado en el proyecto |
| GitHub | Plataforma para alojar, versionar y compartir proyectos |

## Evidencias

- Repositorio creado correctamente.
- README principal visible en GitHub.
- Carpeta `portfolio/midtech/` creada.
- README del proyecto MIDTECH creado.
- Commits registrados en el historial del repositorio.
